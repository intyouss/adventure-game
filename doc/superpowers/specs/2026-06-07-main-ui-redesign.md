# 主界面UI重设计

**日期**: 2026-06-07
**状态**: 已确认

## 概述

将当前TabContainer+底部导航的布局改为垂直堆叠布局，重新组织战斗、装备、开箱区域，并引入技能仓库、商店和排行榜侧滑面板。

## 布局结构（自上而下）

```
┌─────────────────────────┐
│         HUD             │  Lv / EXP / 💰 / 🎫 / CP
├─────────────────────────┤
│     战斗/关卡区域        │  章节标签 · 关卡名居中 · 对战画面 · 5技能槽
├─────────────────────────┤
│      装备区域            │  10格装备槽（5×2网格）
├─────────────────────────┤
│      开箱区域            │  箱子数量 · 区域等级 · 开箱/升级按钮
├─────────────────────────┤
│ 技能仓库 │ 商店 │ 🏆榜  │  底部固定按钮行
└─────────────────────────┘
```

## 各区域详细

### 1. 战斗/关卡区域

- **章节标签**: 水平居中排列，可切换章节
- **关卡名**: 居中大字显示（如 "1-5"），点击直接进入战斗
- **对战画面**: 角色在左、怪物在右，显示双方血条
- **技能槽**: 5个水平排列，固定在战斗区下方
  - 第1位: 职业固定技能（🔥特殊边框标识，不可更换）
  - 第2-5位: 可装备技能槽，无装备时显示空白虚线框(+)
  - 紧凑排列，不显示技能名称和等级

### 2. 装备区域

- 10格装备槽，5列×2行网格
- 槽位: 武器、头盔、铠甲、鞋子、戒指1、戒指2、项链、护腕、腰带、手套
- 已装备显示品级颜色+名称，空槽显示虚线边框+[空]
- 点击装备槽可卸下

### 3. 开箱区域

- 紧凑一行: 箱子数量 | 区域等级 | 开箱按钮 | 升级按钮
- 开箱获得装备后弹出对比弹窗（见下方）

### 4. 底部按钮行

- 三个按钮水平排列，固定显示
- **技能仓库**: 点击后装备区和开箱区隐藏，战斗区保留，显示技能网格
- **商店**: 点击后战斗区+装备区+开箱区全部隐藏，显示商店列表
- **🏆排行榜**: 点击后右侧滑出排行榜面板

## 交互模式

### 技能仓库模式

- 触发: 点击底部"技能仓库"按钮
- 状态: 战斗区继续显示，装备区和开箱区隐藏
- 内容: 技能列表，6列网格，一页4行（24个），可下滑
- 每格显示: 技能名称 + 等级（Lv.x）
- 交互: 点击技能 → 替换到上方对应技能槽
- 返回: 再次点击"技能仓库"按钮或×按钮退出

### 商店模式

- 触发: 点击底部"商店"按钮
- 状态: 战斗区+装备区+开箱区全部隐藏
- 内容: 商店列表，2列网格
  - 技能商店（抽卡/购买）
  - 装备商店（待开放）
  - 其他商店（待扩展）
- 返回: 再次点击"商店"按钮或×按钮退出

### 排行榜面板

- 触发: 点击右侧小按钮（或底部🏆按钮）
- 动画: 从右侧滑出
- 展开后触发按钮隐藏，面板上有收起按钮
- 内容: 章节标签切换 + 排名列表（含我的排名置顶高亮）+ 加载更多

### 开箱装备对比弹窗

- 触发: 开箱获得装备时自动弹出
- 情况1: 对应槽位为空 → 仅显示新装备属性
- 情况2: 对应槽位有装备 → 上下分两部分对比显示
  - 上部: 新装备（品质颜色+属性: ATK/DEF/HP）
  - 下部: 当前装备（属性对比）
- 操作: "保留原装备" / "替换新装备"
- 未被选择的装备自动分解

## 废弃内容

- 移除 TabContainer 和 BottomNav
- 移除独立的技能抽卡按钮（移至商店内）
- 移除装备背包/分解功能
