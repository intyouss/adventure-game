# 冒险大作战 - 游戏需求文档

> 版本：v1.1 | 日期：2026-06-04

---

## 一、项目概述

### 1.1 项目定位

一款放置类休闲 RPG 联网手游，参考《冒险大作战》（Capybara Go!）核心玩法。玩家组建角色、搭配技能与装备，通过自动战斗推进关卡，核心乐趣在于装备收集、技能抽取与通关进度竞争。

### 1.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 客户端 | Godot Engine | 纯客户端，负责渲染、UI、战斗表现、上报结算 |
| 服务端 | Golang | 纯后端，负责账号、存档、战斗校验、装备数据、排行榜 |
| 通信协议 | HTTP RESTful + WebSocket | RESTful 用于登录、存档、开箱、抽技能、排行榜等业务；WebSocket 用于战斗上报与校验 |
| 后端框架 | Gin | Go 高性能 HTTP 框架 |
| 数据库 | PostgreSQL | 利用 JSONB 存储装备/技能列表，支持排行榜查询 |
| 目标平台 | Android | 首发仅 Android |
| 美术 | 占位方块 | 初期用几何图形代替，逻辑优先 |

## 二、账号系统

### 2.1 注册与登录

- 支持**手机号注册/登录**和**邮箱注册/登录**
- 密码加密存储
- 登录后返回 Token，后续请求携带 Token 鉴权

### 2.2 账号数据

服务端持久化存储：
- 玩家基本信息（ID、昵称、等级、经验值）
- 装备数据
- 技能数据
- 关卡进度
- 开箱区域等级
- 货币余额（金币、技能券等）

---

## 三、核心战斗系统

### 3.1 战斗模式

- **自动战斗**：角色自动攻击、移动，玩家无需手动操作普攻
- **主动选技能**：玩家可在技能仓库选择和搭配技能到 4 个可选技能栏，战前自由搭配
- 战斗计算由**客户端执行**，结算后上报**服务端校验**

### 3.2 技能槽位

- **1 个固定职业技能**：每个职业自带一个固有技能
- **4 个可选可更换技能**：玩家从已拥有的技能池中装备 4 个，战前自由搭配

### 3.3 关卡结构

- 每章共 **10 关**（命名如 `1-1` 至 `1-10`）
- 每关包含 **4 波小怪 + 1 个 BOSS**
  - 前 4 波：小怪群
  - 第 5 波：BOSS
- 通关当前章全部 10 关后，进入下一章（`2-1` 开始），难度递增

### 3.4 战斗校验

服务端采用 **方案 A + B**：

- **方案 A（轻量合理性校验）**：检查上报的伤害/通关时间是否在玩家当前战力允许范围内，超出阈值拒绝
- **方案 B（服务端重跑）**：服务端用相同输入（玩家属性 + 技能 + 装备 + 关卡参数）跑简化版战斗逻辑，比对关键指标（是否通关、大致伤害量），超出容差则拒绝

## 四、角色与经验

### 4.1 职业

- **第一版仅 1 个职业**（后续扩展）
- 职业决定固定职业技能和基础属性

### 4.2 等级与经验

- 经验获取途径：**通关关卡**、**装备分解**
- 经验曲线：**指数增长**
- 装备分解：**品质越高的装备，分解获得的经验越高、金币越多**
- 升级提升基础属性（攻击、防御、生命等）

### 4.3 装备分解

- 玩家可分解不需要的装备
- 分解产出：**经验值 + 金币**
- 分解收益与装备品质挂钩，品质越高收益越大

---

## 五、技能系统

### 5.1 技能获取

- 通过**技能商店抽取**（Gacha 机制）
- 消耗货币：**技能券**（Skill Ticket）

### 5.2 技能品质

| 品质 | 颜色 | 稀有度 |
|------|------|--------|
| 普通 | 白色 | 极低 |
| 稀有 | 蓝色 | 低 |
| 精良 | 紫色 | 中等 |
| 史诗 | 橙色 | 高 |
| 传说 | 红色 | 极高 |

- 高品质技能抽取概率随**技能商店等级**解锁

### 5.3 技能商店等级

- 技能商店根据**累计抽取次数**升级
- 商店等级越高，可抽取到高品质技能的概率越大

### 5.4 技能券获取途径

- 通关关卡奖励
- 每日登录赠送
- 充值购买（第一版不做充值，后续加入）

## 六、装备系统

### 6.1 装备栏位（10 个）

| 序号 | 部位 |
|------|------|
| 1 | 武器 |
| 2 | 头盔 |
| 3 | 衣服 |
| 4 | 鞋子 |
| 5 | 戒指 1 |
| 6 | 戒指 2 |
| 7 | 项链 |
| 8 | 护腕 |
| 9 | 腰带 |
| 10 | 手套 |

### 6.2 装备品质

| 品质 | 颜色 | 说明 |
|------|------|------|
| 普通 | 白色 | 基础属性 |
| 优秀 | 绿色 | |
| 稀有 | 蓝色 | |
| 精良 | 紫色 | |
| 史诗 | 橙色 | |
| 传说 | 红色 | |
| 神话 | 彩色/彩虹 | 最高品质 |

- **无强化/升级系统**，装备属性在获得时即确定
- 每个品质档位有**属性数值范围**，同品质装备属性在范围内随机

### 6.3 装备获取

- 唯一途径：**开箱子**
- 箱子来源：**关卡掉落**（每关随机掉落 2-3 个箱子）
- 箱子本身无等级区分

### 6.4 开箱区域等级

- **开箱区域有独立等级**，决定箱子能开出的装备品质上限
- 区域等级越高，高品质装备可抽取概率越大，并解锁更高品质
- 区域等级通过**消耗金币独立升级**

### 6.5 装备分解

- 分解不需要的装备
- 产出：**经验值 + 金币**（品质越高越多）

## 八、数值公式

### 8.1 伤害公式

```
伤害 = 攻击力 × 技能系数 / 防御力
```

- 乘除法模型，攻防数值等比增长时伤害保持稳定
- 第一版先定一套占位数值，后续根据测试调整

### 8.2 经验曲线

- **指数增长**：每升一级所需经验按指数递增
- 初期先定一套基础数值，确保前期升级节奏流畅

---

## 九、排行榜

### 9.1 排名依据

- **通关进度**：按已通关的最高章节排名
- 同进度按通关时间/战力等次级维度排序

### 9.2 后续扩展

- PVP 排行榜
- 战力排行榜
- 赛季制

## 十二、技术决策记录

| 决策项 | 选择 | 理由 |
|------|------|------|
| 通信协议 | HTTP RESTful + WebSocket | RESTful 简洁适合业务 CRUD，WebSocket 适合战斗上报校验 |
| 战斗校验 | 方案 A + B | 轻量合理性校验 + 服务端重跑，平衡性能与安全 |
| 数据库 | PostgreSQL | JSONB 灵活存储装备/技能，排行榜查询性能好 |
| 后端框架 | Gin | 社区成熟、性能好、文档丰富 |
| 伤害公式 | 攻击力 × 系数 / 防御力 | 乘除法，攻防等比增长时伤害稳定 |
| 经验曲线 | 指数增长 | 前期快后期慢，配合装备分解获取经验 |
